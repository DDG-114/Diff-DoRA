# 导师汇报讲稿（口语版）

老师您好，我这段时间围绕“用 LLM 做陕西电力现货日前电价预测”这个方向，做了比较系统的验证，目前已经得到一个比较清楚的阶段性结论。

## 1. 先说目标

我的任务是：

- 用 `2025` 年数据训练
- 预测日前 `96` 个时间点，也就是一天的 `15min x 96`
- 希望最后的日均精度可以达到 `80%`

## 2. 先说最重要的判断

我现在最明确的结论是：

**直接让 LLM 一次性生成 96 个未来电价点，不现实。**

我已经查过相关文献，也结合了我们项目里的实验结果，发现这条路的主要问题是：

1. 输出长度太长，容易解析不稳定
2. 数值型序列对 LLM 不太友好
3. 电价这种数据有地板价、尖峰价、强日内结构，直接生成很容易塌缩成常数或者漂移

所以我后面把路线改成了：

- 先用一个强基线给出整天候选曲线
- 再让 LLM 去修正这条曲线

## 3. 我做了哪些路线

我大概测试了几类 LLM 路线：

1. 直接预测 `96` 点
2. 先预测 `16` 点，再滚动拼成 `96` 点
3. 给 LLM 一个 candidate 曲线，让它做逐点残差修正
4. 只让 LLM 修正 candidate 最不可信的部分
5. 让 LLM 不改整条曲线，只预测“整体应该上移还是下移多少”

## 4. 当前实验最清楚的结果

现在实验已经很明确地说明：

### 第一

LLM 直接做整条曲线主预测器，这条路不合适。

### 第二

candidate baseline 本身非常重要，它现在仍然是最稳的主体。

### 第三

如果一定要保留 LLM，当前最有前景的方式不是让它去重写整条曲线，而是：

**让它做日级别的 offset 校准。**

也就是：

- candidate baseline 负责整天 96 点曲线的形状
- LLM 只负责判断今天整体价位应该上移还是下移多少

## 5. 当前最好结果到哪里了

如果按我现在最接近目标的 LLM 路线，也就是“candidate + LLM 日级 offset”来看，

当前跑出来的大致结果是：

- `mean_daily_mean_accuracy ≈ 0.795`

也就是说：

**已经非常接近 0.8 了，但还没有严格超过 0.8。**

## 6. 这意味着什么

这说明目前课题的研究状态不是“做不通”，而是：

- 路线已经做通了
- LLM 已经能稳定参与预测
- 但是当前最稳妥的角色，不是主预测器，而是校准器

换句话说，现在比较合理的结论是：

> LLM 在这个任务里更适合作为 candidate baseline 的辅助修正模块，而不是单独承担整条 96 点曲线预测。

## 7. 我认为下一步最值得做什么

如果继续做这个方向，我觉得最值得继续的不是再让 LLM 去逐点生成，而是：

1. 继续优化 candidate baseline
2. 让 LLM 只做日级 offset 校准
3. 或者只在 candidate 最不可信的时间段做局部修正

## 8. 当前阶段结论

所以我现在的阶段性结论是：

1. 直接用 LLM 预测 96 点电价曲线，不现实
2. candidate + refinement 是合理方向
3. 当前最接近目标的 LLM 方案，是“日级 offset 校准”
4. 目前结果已经逼近 `80%`，但还没有严格超过

如果后续继续推进，我建议就围绕这条“candidate baseline + LLM 日级 offset”路线继续做，而不是再分散到逐点生成路线。
